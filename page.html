  • OzoneWebmail Funcional Tests: Escrito em Python e usando Selenium Webdriver + Unittest, usamos um servidor para poder ser acessível remotamente de outros ambientes, como desenvolvimento, preview e homologação. Integrado ao Jenkins, os testes são rodados antes de ir para produção. Como ambientes de preview as máquinas são Centos(py2.6) e desenvolvimento Ubuntu(py2.7/3). Foi criado um pequeno script bash para gerenciamento da execução dos testes funcionais.
  • delivery-mod-ozone Arquitetura OzoneWebmailTemplates: Trabalhamos com os arquivos frontends separados do backend, foi necessário criar um módulo para buscar templates de lugares diferentes, não ficando mais atrelado ao serviço de entrega. Com isso pudemos juntar os templates de backend junto com os sources de frontend. O benefício vem que antes era necessário fazer dois deployes, um para cada projeto, sempre tendo problemas de um tempo pequeno de quebra. Esse módulo permite ter tudo em um único commit.
  • Ozone Webmail: Nova interface para o Email do Terra, com o conceito single page application, precisamos desenvolver uma robusta api de comunicação com o servidor de email. Nunjuncks para renderização e componentização, zaz-framework(framework frontender próprio) que nos permite escrever JavaScript puro e ele resolve os problemas de implementação. Produto está em produção para Argentina, Colombia, Chile, Peru, Venezuela, EUA, México. Agora estamos trabalhando para fazer a virada no Brasil.
  • Terra Chat: Substituimos tecnologias Ejabberd, XMPP e Erlang, migrando para Firebase e fluxos próprios escrito em Python.
  • Terra Serviços: Reformulação do visual da vitrine virtual do Terra. No meio do processo desenvolvemos widgets para podermos integrar em qualquer lugar do Terra, basicamente os widgets são módulos/apps JavaScript consumindo serviços.
  • StickAd: Solução FontEnd voltado para aumento das visualizações das publicidades, não podendo revelar o ganho mas um simples JavaScript teve ganhos significativos chegando a mais de 30% available.
